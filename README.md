# Baby Sleep Tracker

Веб-приложение для трекинга сна, бодрствования и замеров роста/веса ребенка.

## Основные функции

- **Профиль ребенка**
  - создание профилей
  - переключение активного ребенка
- **Сон**
  - быстрый старт/стоп сна (дневной/ночной)
  - отображение текущей длительности сна и ВБ
  - проверка пересечений интервалов сна
- **День**
  - список записей сна за день или за последние 24 часа
  - ручное добавление, редактирование и удаление записей
  - теги и заметки для каждого сна
- **Статистика**
  - выбор даты
  - общий сон, общее ВБ, средние значения
  - время начала дня и ухода в ночной сон
- **Рост и вес**
  - добавление замеров роста/веса
  - график по возрасту (упрощенные кривые P3/P50/P97)
  - история измерений
- **Экспорт**
  - выгрузка в `.xlsx` (листы: `Children`, `Sleep`, `Growth`)

## Технологии

- Next.js 14 (App Router) + TypeScript
- IndexedDB (Dexie)
- xlsx

## Как развернуть локально

Требования:

- Node.js 20+
- pnpm 9+

Установка и запуск:

```bash
pnpm install
pnpm dev
```

Открыть в браузере: `http://localhost:3000`

## Сборка

```bash
pnpm build
```

Проект настроен на статический экспорт (`output: 'export'`), после сборки сайт лежит в папке `out/`.

Локальный предпросмотр статической версии:

```bash
npx serve out
```

## Деплой

### GitHub Pages

Репозиторий уже содержит workflow: `.github/workflows/deploy.yml`.

Достаточно пуша в `main`:

1. Сборка запускается в GitHub Actions.
2. Артефакт из `out/` публикуется в GitHub Pages.

`basePath` для project pages выставляется автоматически в `next.config.js`.

Если нужен другой префикс пути, можно задать переменную:

- `NEXT_PUBLIC_BASE_PATH`

## Хранение данных

Все данные хранятся локально в браузере (IndexedDB).
Серверная синхронизация и обязательная авторизация в текущей версии не используются.
